Defines interfaces to defines a Schema. The definition is geared to describe persistent data in a key-value database. Hence, type definition describes which attributes participate in forming the primary key, or an attribute can be qualified to represent a secondary index.
  • Schema is a set of Types that can reference each other.
  • Each Type has zero or more named attributes
  • Each Attribute is of certain type that determines what value can be held in that attribute. An Attribute can also other qualifiers such as whether it is indexed, its cardinality or whether it allows null values etc.
