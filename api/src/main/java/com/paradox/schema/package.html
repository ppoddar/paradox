Interfaces for Schema or a Type system.
These definitions describe a type system that can be stored in a database. Hence, type definition can have constraints on attribute(s) to form primary key, or secondary index.
  • Schema is a group of related Types.
  • Types are categorized as: Primitive, Enumerated, Container and User-defined
  • Each user-defined type has zero or more named attributes
  • Each Attribute is of certain type that determines what value can be held in that attribute.
  • Constraints are set on one or more attribute to qualify or restrict whether it is indexed, its cardinality or whether it allows null values etc.
  • Schema imposes a closure on the types. The type of each attribute is also a type contained in the same schema.
