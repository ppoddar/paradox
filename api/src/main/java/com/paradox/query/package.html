Interfaces to a Query Engine for Oracle NoSQL store. This package provides the user interfaces to execute a query using a SQL like string. A query engine on a key-value store requires few essential artifacts that must be supplied by the user of this API. This package defines these artifacts as following interface contracts:
  1. {@link com.oracle.nosql.query.KeyMaker KeyMaker}
  2. {@link com.oracle.nosql.query.ValueTransformer}
  3. {@link com.oracle.schema.Schema}
  4. and last, but not the least, {@link oracle.kv.KVStore}
A concrete implementation is used in client code, as follows, to execute a query: 
   
       KVStore store = // a connection to Oracle NoSQL store
       QueryContext ctx = new QueryEngine().setStore(store)
                                           .setValueTransformer(new JSONValueTransformer())
                                           .setSchema("person-address-schema.json");
       String sql = "select * from Person where name=:name and age > :age"; 
       List result = ctx.bindParameter("name", "John")
                                    .bindParameter("age", 25)
                                    .executeQuery(sql);
The query engine accepts {@code SELECT} query string following a SQL-like syntax. The query string is parsed by an ANTLR-generated parser. If a {@link com.oracle.schema.Schema schema} is available, then the query is also checked for semantic validity i.e. all name tokens (the candidate type, the field paths) must be present in the schema.
